Release Notes for 360 | SpeechAnywhere Services 2.1 Service Release 4 (Android edition)

New in service release 4

  • Support for 64-bit applications.

Solved issues in service release 4

  • The undo that voice command no longer moves the cursor to the wrong place. [DNB-4944]
  • Application commands defined using an escaped quotation mark (\") no longer cause an error. [DNB-4759]
  • Calling synchronize() from the event handler for OnProcessingFinished or OnProcessingFinishedForControl no longer causes the application to stop responding. [DNB-4760]
  • The first speech-enabled control no longer automatically receives the UI focus when the VuiController is opened and synchronized. [DNB-5080]
  • Closing the VuiController while recognition results are pending no longer causes the application to stop responding. [DNB-5145]

Important information

  • 360 | SpeechAnywhere Services 2.1 (Android edition) is compatible with server versions 1.6, 2.0 and 2.1.
  • 360 | SpeechAnywhere Services 2.1 (Android edition) will not work correctly when used with server version 1.5.
  • 360 | SpeechAnywhere Services 2.1 Server is compatible with client versions 1.5, 1.6, 2.0 and 2.1.
  • To use Bluetooth headsets you must set android.permission.BROADCAST_STICKY, android.permission.BLUETOOTH and android.permission.MODIFY_AUDIO_SETTINGS within the <manifest> tag.
  • SpeechAnywhere Analytics has been renamed to Dragon Medical Analytics.

New in version 2.1

  • The Android SDK is now optimized for the Android Studio integrated development environment.
  • Custom control implementations are now supported. For more information, see: 'Custom controls' in the SpeechAnywhere SDK help.
  • The handling of temporary network problems has been improved.
  • Continuous recording across VUI form boundaries is now possible.
  • The integrator can now actively improve speech recognition; notifying SpeechAnywhere about the document correction state as well as the finalized version via the support functionality for document workflows. For more information, see: 'Support functionality for document workflows' in the SpeechAnywhere SDK help.

Known issues

  • Recording and speech recognition do not work with an Android emulator. [PSP00073974]
  • For SpeechMagic-based systems the SpokenText parameter in the CommandRecognized event is empty in command mode. [PSP00062085]

  • The speech bar covers part of the soft keyboard if your Activity has its theme set to "android:theme="@android:style/Theme.Dialog" in the AndroidManifest.xml file. For the speech bar to be positioned correctly, you must set windowSoftInputMode to stateHidden in the activity tag. For example:

    <activity

    android:windowSoftInputMode="stateHidden"

    android:theme="@android:style/Theme.Dialog"

    ...

    </activity>

    Once this setting is made, the speech-enabled activity will start with the keyboard hidden.

  • Android devices 4.3. or lower: The speech bar flickers when the system starts or stops processing. [DNB-4260]
  • The speech bar is always displayed in the foreground, even when the user is interacting with a list or a picker. [DNB-5075]

Infrastructure

Infrastructure no longer supported

  • Version 2.1 client applications do not work with version 1.4 servers.
  • Android OS version 4.0 or higher will be required.

Infrastructure changes planned for the next release

  • In 360 | SpeechAnywhere Services 2.2 all version 1.5 and 1.4 client-based applications will stop working.
  • In 360 | SpeechAnywhere Services 2.2 all version 2.2 client-based applications will stop working with version 1.5 servers.

*End-of-life: third party environments

Support for third party environments is only valid as long as they are supported by the corresponding vendor and might be subject to other restrictions. Please contact Nuance Technical Support for details. For more information, see the documentation delivered with the third-party product and supporting Nuance documentation.

When support by the vendor has stopped, Nuance will continue support if an issue is specific to the Nuance solution, within the limitations of the vendor’s and Nuance's end-of-life policy. This means that issues that are a combination of the vendor’s component and the Nuance solution cannot be supported.
