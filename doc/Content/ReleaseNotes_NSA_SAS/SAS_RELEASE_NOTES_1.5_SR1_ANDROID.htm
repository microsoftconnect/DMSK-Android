Release Notes for 360 | SpeechAnywhere Services 1.5 Service Release 1 (Android edition)

Solved issues

The speech bar no longer flickers when an Activity is themed as a dialog. [PSP00072311]

The Android edition now displays an error message if an SAS Service machine is out of resources. [PSP00072052, PSP00073396]

Password text controls are no longer speech-enabled by default. [PSP00072770]

When an Activity is unloaded in the background by the Android runtime, bringing it to the foreground again no longer fails and the integration does not stop unexpectedly. [PSP00071579]

The speech bar now no longer opens over the keyboard when an Activity is themed as a dialog; you must set windowSoftInput to stateHidden in the activity tag. [PSP00073401]

Text controls which are not descendants of the VuiController are now no longer speech enabled. [PSP00073442]

The What You Can Say page is now displayed displayed at the correct size even if the Activity does not fill the screen of the device. [PSP00073437]

Following the recent Apple iOS security issue (http://support.apple.com/kb/HT6147) we conducted a thorough review of the 360 | SpeechAnywhere Services Android SDK and identified a potential vulnerability. This security issue could make an application in an on-premise deployment vulnerable to man-in-the-middle attacks. At this time we have no reason to suspect this vulnerability has been exploited; however, we recommend that all partners upgrade to this new SAS SDK to ensure the highest levels of security. [PSP00073528]

Important information

  • 360 | SpeechAnywhere Services 1.5 Server is compatible with client versions 1.3, 1.4 and 1.5.
  • The speech bar covers part of the soft keyboard if your Activity has its theme set to "android:theme="@android:style/Theme.Dialog" in the AndroidManifest.xml file. For the speech bar to be positioned correctly, you must set windowSoftInputMode to stateHidden in the activity tag. For example:

<activity

android:windowSoftInputMode="stateHidden"

android:theme="@android:style/Theme.Dialog"

...

</activity>

Once this setting is made, the speech-enabled activity will start with the keyboard hidden.

Known issues

  • In landscape mode, on the devices Samsung Galaxy Nexus, Nexus S, Note 2, HTC Desire HD, HTC one X, Sony XPERIA, Motorola Milestone 2, the speech bar is not positioned properly due to the Android Framework bug 20235. Workaround: Use portrait mode. [PSP00058627]
  • For SpeechMagic-based systems the SpokenText parameter in the CommandRecognized event is empty in command mode. [PSP00062085]
  • In landscape mode, on the devices Sony XPERIA, Motorola Xoom, when the Manage Vocabulary page is displayed, it is currently not possible to scroll to the text control and enter a word. Workaround: Use portrait mode. [PSP00068838]

  • If the field with the focus is removed from the VUI form, the first field will get the focus after calling synchronize(). [PSP00068978]

Infrastructure

Infrastructure no longer supported

  • Version 1.1 client-based applications will stop working.

Infrastructure changes planned for the next release

  • In 360 | SpeechAnywhere Services 1.6 all version 1.2 client-based applications will stop working.

*End-of-life: third party environments

Support for third party environments is only valid as long as they are supported by the corresponding vendor and might be subject to other restrictions. Please contact Nuance Technical Support for details. For more information, see the documentation delivered with the third party product and supporting Nuance documentation.

When support by the vendor has stopped, Nuance will continue support if an issue is specific to the Nuance solution, within the limitations of the vendor’s and Nuance's end-of-life policy. This means that issues that are a combination of the vendor’s component and the Nuance solution cannot be supported.
