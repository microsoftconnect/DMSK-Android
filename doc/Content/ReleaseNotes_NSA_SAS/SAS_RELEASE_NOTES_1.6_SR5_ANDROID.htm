Release Notes for 360 | SpeechAnywhere Services 1.6 Service Release 5 (Android edition)

Solved issues in service release 5

Text input issues after changes to the screen orientation in the Manage Vocabulary page and the help have been solved. [PSP00068302, PSP00068716]

The What You Can Say page has been made more responsive. [PSP00078030]

The What You Can Say page now loads correctly after the user switches to another speech-enabled activity. [PSP00077922]

Voice commands that select the current text (for example, select this word) now work correctly after the cursor has been moved within the text. [PSP00073388]

Voice commands that select text now work correctly when several are used consecutively. [PSP00076490]

The undo that and redo that voice commands now work correctly in situations where selected text has been overwritten or where the cursor has been moved within existing text before recognition. [PSP00076041, PSP00076043]

When text is inserted within existing text or over selected text, the surrounding text is correctly formatted. [PSP00075130]

The right swipe gesture no longer opens the keyboard. [PSP00074838]

Important information

  • 360 | SpeechAnywhere Services 1.6 Server is compatible with client versions 1.4, 1.5 and 1.6.
  • 360 | SpeechAnywhere Services 1.6 (Android edition) is compatible with server versions 1.4, 1.5 and 1.6.
  • To use Bluetooth headsets you must set android.permission.BROADCAST_STICKY and android.permission.BLUETOOTH within the <manifest> tag.

New in version 1.6

  • Voice editing functionality has been extended with select & say voice commands (Dragon-based systems only).
  • SAS clients can now use Dragon customization packs (.dat files) containing Text-and-Graphics/auto-texts without lists. You can import and share .dat files via the Nuance Management Console.
  • The Android edition now supports Bluetooth devices.

Known issues

  • Recording and speech recognition do not work with an Android emulator. [PSP00073974]
  • For SpeechMagic-based systems the SpokenText parameter in the CommandRecognized event is empty in command mode. [PSP00062085]

  • If the field with the focus is removed from the VUI form, the first field will get the focus after calling synchronize(). [PSP00068978]

  • The speech bar covers part of the soft keyboard if your Activity has its theme set to "android:theme="@android:style/Theme.Dialog" in the AndroidManifest.xml file. For the speech bar to be positioned correctly, you must set windowSoftInputMode to stateHidden in the activity tag. For example:

    <activity

    android:windowSoftInputMode="stateHidden"

    android:theme="@android:style/Theme.Dialog"

    ...

    </activity>

    Once this setting is made, the speech-enabled activity will start with the keyboard hidden.

Infrastructure

Infrastructure no longer supported

  • Version 1.2 client applications will stop working with version 1.6 servers.

Infrastructure changes planned for the next release

  • In 360 | SpeechAnywhere Services 1.7 all version 1.3 client-based applications will stop working.

*End-of-life: third party environments

Support for third party environments is only valid as long as they are supported by the corresponding vendor and might be subject to other restrictions. Please contact Nuance Technical Support for details. For more information, see the documentation delivered with the third-party product and supporting Nuance documentation.

When support by the vendor has stopped, Nuance will continue support if an issue is specific to the Nuance solution, within the limitations of the vendor’s and Nuance's end-of-life policy. This means that issues that are a combination of the vendor’s component and the Nuance solution cannot be supported.
