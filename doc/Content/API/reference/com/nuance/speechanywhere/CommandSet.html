Class Overview

Representation of an application-defined command set.

A command set is a group of application commands that logically belong together. It is characterized by a human-readable name and a description that will appear on the personalization & help page.

Command sets contain a set of application commands. Each command must have a unique ID that identifies the recognized command in the onCommandRecognized(String, String, String, HashMap) callback invocation. Commands should have a human-readable description and a spoken form. CommandPlaceholders can be used in spoken forms.

A command set can be enabled (default) or disabled, in which case all the commands it contains are disabled.

Do not manipulate the fields of the CommandSet object directly - use the createCommand(String, String, String, String), enableCommand(String, boolean), setEnabled(boolean) methods instead.

public void createCommand (String id, String phrase, String displayString, String description)

Adds a new application command to the command set.

Parameters
id The unique identifier of the command. Make sure the application command ID does not contain spaces and does not start with a number.
phrase The spoken form of the command.
displayString The spoken form of the command as displayed on the help page.
description The human-readable description of the command.

Can contain CommandPlaceholder identifiers: in this case, the corresponding placeholder objects must be created and supplied to the VuiController in a assignCommandPlaceholders(CommandPlaceholder[]) method call.

Placeholder identifiers must be enclosed in < and > in the spoken form. For example, if the ID of the referenced placeholder is "patient", a correct spoken form can be "select patient <patient>"

To set alternative spoken forms, call this method repeatedly, once for every alternative.
Throws
InvalidParameterException if id or phrase is null or empty.
