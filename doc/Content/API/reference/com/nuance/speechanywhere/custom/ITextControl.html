public abstract String getId ()

Returns a unique identifier for the user-visible UI element this custom control instance is associated with.

If your ITextControl implementation is destroyed and recreated across the configuration changes of your Activity (like the Views are), Dragon Medical SpeechKit will automatically match the new ITextControl instances to the old ones so that speech recognition can operate uninterrupted. However, this identifier has to be persistent and unique across all custom controls in your application. A sensible value which could be used here is the Android ID of the View this custom control is associated with.

Returns
  • The unique ID.

public abstract boolean isFocused ()

Returns whether this control has the focus or not.

The "focus" concept implemented may differ from the Android GUI focus concept. In particular, it is recommended to keep track of the last focused speech-enabled control to avoid having all controls lose focus when the end user clicks on a control of another kind (for example, the "Start recording" button). Make sure exactly one control returns true at any given time.

public abstract void replaceText (int start, int length, String newText)

Replaces a portion of the text in this control with a new string.

After the call the insertion point must be located at the same position in the text as before the call. This means that if ReplaceText() inserts or removes characters to the left of the current insertion point or selection, the location of the insertion point must be moved accordingly. If ReplaceText() modifies text to the right of the insertion point, the insertion point must not change. If ReplaceText() modifies characters at the insertion point, the insertion point must be set to the end of the modified region. Ideally, if ReplaceText() is called while the user is typing and modifies text to the left or right of the typing location, this will not interfere with the user's keyboard input. This is to ensure that speech recognition results can be inserted in the text asynchronously while the user is, for example, correcting a misrecognition at another location in the text.

Parameters
start Zero-based index of the first character to be replaced.
length Length of text to be replaced.
newText String containing the new text.
