Class Overview

Controller managing the voice user interface (VUI) form attached to a graphical user interface (GUI) activity.

This controller manages the VUI aspects of a GUI activity and speech-enables your custom control implementations. The application is responsible for keeping the VUI in sync with subsequent changes to the GUI view hierarchy by calling #synchronize().

The VuiController instance can be created either declaratively, in the layout XML file of the activity, or programmatically.

public void controlGotFocus (ITextControl control)

This method should be called by custom control instances whenever one of them receives focus. The parameter is the newly focused ITextControl instance itself.

It is optional to call this method. Dragon Medical SpeechKit will work correctly even if it is not called; however, calling it allows Dragon Medical SpeechKit to detect focus changes more precisely.

public void synchronize (List<Object> controls)

Forces synchronization of the VUI form: updates the set of speech enabled controls and applies the changes in properties.

This method synchronizes the VUI form, controlled by this VUI controller, with the controls passed in to the method. You must call this method to trigger synchronization when the set of speech-enabled controls are to be changed (e.g. you intend to add or remove controls), as well as when any other property of the voice user interface is to be changed (e.g. you have assigned new application commands).

Parameters
controls The list of controls to be speech enabled. It can contain either natively supported android.view.View objects, or ITextControl objects implementing your custom controls.
